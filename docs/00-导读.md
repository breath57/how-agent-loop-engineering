# Loop Engineering 专题导读：这一期，我们把"让 AI 自己把事做完"这件事彻底搞明白

你有没有这种感觉？

AI coding agent 越来越能干了，但它干完一步，你得马上接上下一句。像什么呢？像你雇了个实习生，能力不错，但每干完一件事就得跑来问你："然后呢？"

这个专题要聊的，就是怎么把这个"然后呢"从你脑子里拿走，放进系统里。

**8 篇文章，一条线串下来。**

---

## 专题里有什么

| 篇 | 标题 | 一句话说清楚 |
|---|---|---|
| 01 | 从问题出发 | 为什么我们需要让 AI 自己"循环干活" |
| 02 | Loop 的最小形态 | 一个 bash while 循环怎么就成了 Loop 的原型 |
| 03 | 状态管理 | Agent 健忘怎么办？用文件当它的笔记本 |
| 04 | 验证机制 | Agent 说"做完了"你敢信吗？怎么让它拿证据说话 |
| 05 | 多 Agent 协作 | 一个 Agent 不够用的时候，怎么让几个 Agent 接力干活 |
| 06 | 安全与人机协作 | 哪些事能放手，哪些事必须人来拍板 |
| 07 | 落地模板 | 从零搭一个能跑的 Loop，照着做就行 |
| 08 | 续集：别让它骗自己 | Loop 最危险的不是跑不完，而是跑得挺嗨但啥也没干成 |

---

## 怎么读

建议从 01 开始按顺序读。

前两篇是"为什么要这么做"，帮你建立直觉。
中间四篇是"具体怎么做"，拆解核心机制。
最后两篇是"怎么落地"，给你可以直接抄的模板和必须警惕的坑。

如果你已经对 Loop Engineering 有概念，直接跳 04 验证机制也行。那篇是整个专题最核心的一篇。

---

## 这个专题不打算做什么

不打算泛泛而谈"AI Agent 是未来"。

不打算列一堆框架名字让你自己去查。

每一篇都会落到具体的设计选择、代码示例、和实际踩过的坑。

你读完应该能自己设计一个 Loop，而不只是知道有这么个东西。

---
